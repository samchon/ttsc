The compiler-resolved TypeScript graph for a relationship or code-flow question. Reach for it first, before shell/grep/read.

One broad fuzzy query (owner + action + nouns, e.g. "repository find manager query builder") returns the matched declarations with their calls, callers, types, blast radius, and source: the whole cluster in one call, so you do not query one symbol at a time.

Answer from the result; its edge targets are already part of the path, so a node shown as an edge target is part of the answer, not a reason to re-query or grep. After you edit a file, query again rather than reuse an old result.
