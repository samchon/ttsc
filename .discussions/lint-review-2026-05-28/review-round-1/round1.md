# Round 1 — Opening Statements

Lead (scribe): each agent names the most damaging issue in their slice plus a one-line context.

## Agent 1 — linthost/ rule implementations

Top finding is **`rules_unicorn_no_useless_collection_argument.go:37-40`** — the early-return path reports on zero-argument `new Set()` / `new Map()`. That's the canonical correct shape; the rule was meant to flag *present-but-useless* initializers. This currently fires on every `new Set()` and `new Map()` in user code. The diagnostic message ("Don't pass a useless initializer") even contradicts the code path it takes. Adjacent — at least seven other unicorn rules have correctness bugs: `no-unreadable-array-destructuring` uses a threshold of 3 holes where upstream uses 2; `no-negation-in-equality-check` strips parens before checking the `!`, defeating its own documented escape hatch; `prefer-includes` and `consistent-existence-index-check` apply symmetric-operator handling to asymmetric `<` / `>=` / `>`; `consistent-assert` has the wrong body entirely (flags `equal → strictEqual` while the file name is `consistent-assert`); `no-useless-length-check` lists `.every` in the `&&` set even though `every` returns `true` on empty arrays; `no-useless-fallback-in-spread` flags call-argument spreads where the fallback IS load-bearing. Three unicorn rules carry phantom upstream-docs URLs (`no-array-reverse`, `no-array-sort`, `no-immediate-mutation`) that resolve to 404. I also have ~16 medium-confidence findings on missing options and ~5 perf concerns.

## Agent 2 — linthost/ tests

The biggest structural problem is the **boilerplate doc-comment cluster — 161 test files across 10 families** share the same three-paragraph header that says nothing case-specific, violating AGENTS.md §2.2 ("a short paragraph stating the non-obvious *why*"). Distribution: control-flow 35/43, typescript 25/81, runtime-safety 25/29, functions-classes 24/30, arrays-objects 13/14, variables-assignments 11/12, strings-regex 11/17, style-suggestions 10/18, imports-modules 4/8, comments-directives 3/4. The newer per-family work (unicorn, react, cypress, playwright, …) has individually authored doc comments and sets the bar. Second-order: ~70 react/jsx-a11y tests + 12 vitest tests assert findings but **don't assert the rule name**, so a rule-id rename would slip through. `assertFunctionalFinding` uses a "first match wins" pattern that silently accepts multi-finding regressions (~27 functional tests). Four `*_fix_test.go` files put 3 scenarios into one `Test*` function, violating "one test case per file". Four pairs of functional tests are pure duplicates with cosmetic differences. Option-heavy rules (`prevent-abbreviations`, `expiring-todo-comments`, `catch-error-name`, `eqeqeq`, `no-console`, `complexity`, `max-params`) have only default-arm coverage. `unicorn/filename-case` has zero tests.

## Agent 3 — src/ TypeScript

Scope note first: the prompted subdirs (`command/`, `engine/`, etc.) don't exist — all CLI/engine/fix logic is Go. The TS surface is `index.ts` (927 lines) + `defaultFormat.ts` + `structures/`. Within that scope, the **most consequential bug is `unwrapDefault` walking 8 default-hops vs `extractPluginSource` walking 4** — same purpose, side-by-side, asymmetric. One path will silently misroute on the same input. Adjacent: `findLintConfigFile` does 14 `existsSync + statSync` pairs per directory level (effectively 28 syscalls per level); a single `readdirSync` + set intersection would do. The ttsx loader writes raw JSON to stdout with no sentinel — any stray banner from ttsx will turn the parent `JSON.parse` into an opaque error. The default export `createTtscPlugin` is marked `@internal` even though it's the only documented host hook. `boundaries/dependencies` is declared as a rule but missing from `ITtscLintRuleOptionsMap`. `defaultFormat.ts` JSDoc has capitalized `Import` / `Export` from a botched format pass.

## Agent 4 — rule/, lib/, plugin/, lsp.go

The **severity-constant unchecked cast in `contrib_adapter.go:98` (`rule.Severity(ctx.Severity)`)** is the highest-leverage foot-gun: both `rule.go` and `linthost/config.go` independently define `Off=0, Warn=1, Error=2` via `iota`, and no test pins the equality. A reorder of either set silently misroutes severity for every contributor rule. After that: `astutil.NodeText` documents only leading-trivia stripping but also calls `strings.TrimRight` — undocumented and surprises an autofix that wanted to splice exactly at the end. `TokenRange` lacks a `pos >= end` guard so callers can get a zero-width range and a silent no-op fix. `plugin/main.go` lists six subcommands in its banner; the actual dispatcher accepts eleven. The LSP code-action verb threads `--range-json` through three layers and **never reads it** — callers may believe code actions are scoped to a selection when they are not. `copyLSPCommandWorkspaceEntry` uses `defer delete(seenDirs, realDir)` so the symlink-loop guard removes the marker on return, defeating loop detection across sibling links. `RunLSPCommandIDs` / `RunLSPCodeActionKinds` pay the contributor-registration cost on every invocation even though the response is static. The `FixReporter` doc comment contradicts itself ("contributors do NOT implement this interface" → "a contributor authoring a fake reporter for unit tests can declare…"). `go.mod` requires `go 1.26` with no `toolchain` directive, so Go-1.25 contributors get a cryptic error.
