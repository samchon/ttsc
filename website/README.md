# `@ttsc/website`

The `ttsc.dev` site. Built with Next.js 15 (App Router, static export) + Nextra 4 + Tailwind v4.

## Three sections

- **Docs**: Guide documents for consumers, plugin authors, lint users, and editor users (`src/content/docs/`).
- **Blog**: MDX articles under `src/content/blog/`, rendered by the `src/app/blog/` routes; `build/blog.cjs` generates the RSS feed at build time.
- **Playground**: Monaco-based in-browser editor backed by the full Go compiler as wasm, with the banner, paths, strip, lint, and typia plugins linked in.

## Commands

```bash
pnpm install
pnpm dev       # compiler worker rebuild + next dev
pnpm build     # static export into ./out
pnpm deploy    # publish ./out to gh-pages
```

## Playground compiler

The playground keeps typia's worker-boundary architecture (tgrid `WorkerConnector` on the main thread, `WorkerServer` in the worker), with the backend supplied by [`@ttsc/playground`](https://github.com/samchon/ttsc/tree/master/packages/playground).

`build/compiler.cjs` produces the three artifacts the playground loads at runtime:

1. **`public/compiler/playground.wasm`**: the website's consumer wasm, built from the Go module in `website/compiler/` (`cmd/playground`). It links the real Go engines for `@ttsc/banner`, `@ttsc/paths`, `@ttsc/strip`, `@ttsc/lint`, and typia.
2. **`public/compiler/wasm_exec.js`**: Go's bootstrap shim, copied from `@ttsc/wasm`.
3. **`public/compiler/index.js`**: the rspack-bundled Web Worker from `src/compiler/index.ts`, which wires the site's URL conventions and the typia source pack into `createWorkerCompiler` from `@ttsc/playground`.

`build/typia-types.cjs` packs typia's declaration sources into `src/compiler/typia-types.json` so Monaco's `addExtraLib` sees the same type surface the compiler does.
