# Introduction

![banner of ttsc](/og.jpg)

A TypeScript-Go toolchain.

- **`ttsc`** — the compiler. Drop-in for `tsc`. Build, check, watch.
- **`@ttsc/lint`** — replaces `eslint` and `prettier`. Lint violations come out of the compile pass as `error TSxxxxx`; the built-in formatter applies via `ttsc format`.
- **`ttsx`** — runs a TypeScript entrypoint after a real type-check. Drop-in for `tsx` / `ts-node`.
- **`@ttsc/vscode`** — VS Code extension. Lint underlines and plugin diagnostics live in your editor.

Same `tsconfig.json` you already have. Same flags as `tsc`. Five minutes of setup.

```bash
npm install -D ttsc @ttsc/lint @typescript/native-preview
npx ttsc --noEmit
```

## The shape of the project

| | What |
| -- | ---- |
| `ttsc` | The compiler. Build, type-check, watch, fix, format. |
| `@ttsc/lint` | Linter (140+ rules) + Prettier-style formatter. Lint as compile errors; format via `ttsc format`. |
| `ttsx` | TypeScript runner with a real type-check. Typed replacement for `tsx` / `ts-node`. |
| `@ttsc/vscode` | VS Code extension. Lint and plugin diagnostics live as you type. |
| `@ttsc/unplugin` | Same plugin pass inside Vite, Webpack, Rollup, esbuild, Rspack, Rolldown, Farm, Next.js, Bun. |
| `@ttsc/wasm` | The same compiler, in the browser. Build playgrounds with your own plugins linked in. |
| `typia` / `nestia` | Ecosystem plugins. Runtime validators from your types (`typia`); NestJS routes, OpenAPI, SDK (`nestia`). |
