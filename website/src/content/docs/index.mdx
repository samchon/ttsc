# Introduction

![banner of ttsc](/og.jpg)

`ttsc` is a TypeScript-Go toolchain. It runs on `@typescript/native-preview` and works as a drop-in for `tsc`. Plugins like `@ttsc/lint` and `typia` run in the same pass, so your editor, your build, your runtime, and CI all see the same diagnostics.

```bash
npm install -D ttsc @typescript/native-preview
npx ttsc --noEmit
```

Same `tsconfig.json` you already have. Same flags as `tsc`.

## The shape of the project

| | What |
| -- | ---- |
| `ttsc` | The compiler. Build, type-check, watch, fix, format. |
| `ttsx` | TypeScript runner with a real type-check. The typed replacement for `tsx` / `ts-node`. |
| `@ttsc/unplugin` | Same plugin pass embedded inside Vite, Webpack, Rollup, esbuild, Rspack, Rolldown, Farm, Next, and Bun. |
| `@ttsc/lint` | The lint engine. 140 rules. Violations come out as `error TSxxxxx` — same shape as type errors. |
| `typia` / `nestia` | Ecosystem plugins. Runtime validators from your types (`typia`); NestJS controllers + SDK (`nestia`). |

## Reading paths

- **"I just want to use it."** → [Setup](/docs/setup).
- **"I want lint and format as part of my build."** → [@ttsc/lint](/docs/lint). This section is large; the chapter index is the right starting point.

- **"Something broke."** → [Troubleshooting](/docs/troubleshooting) or [FAQ](/docs/faq).
