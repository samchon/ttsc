# Code Graph (MCP)

`@ttsc/graph` has its own benchmark, separate from the compiler timings above.

It asks two questions: how completely the graph resolves a real TypeScript codebase, and whether an agent spends fewer tokens and tool calls when a graph MCP server is available. The agent-cost runner can measure `@ttsc/graph`, `codegraph`, and `codebase-memory` against the same empty-MCP baseline.

The dashboard puts the shared onboarding prompt first: each model gets one chart with every project. Project-specific prompts follow: each project gets one chart with every measured model.

The graph fuses static structure and live diagnostics into one checker-resolved ontology. An agent can see a change's reach over what is currently broken before it edits. See [Code Graph](/docs/graph).

## Agent Cost

The agent-cost benchmark runs an AI coding agent headless against prepared fixtures in `experimental/benchmark/.work`. A baseline run uses an empty MCP config. A graph run uses the same prompt and model, but adds exactly one MCP server: `@ttsc/graph`, `codegraph`, or `codebase-memory`. A graph-arm run that completes without any MCP tool call, or that falls back to shell source reads/searches, is invalid for graph measurement and is retried or left failed rather than used in the median; a cell with no valid sample is not published.

Baseline cells can be measured first with `--arm=baseline --tools=baseline`. Later comparator runs can use `--arm=graph --tools=ttsc-graph,codegraph,codebase-memory`; the website groups those graph samples with the existing baseline by question mode, repository, model tier, effort, fixture branch, and daemon mode.

The stored metrics are the raw per-run samples: total tokens, reasoning token counts when the harness exposes them, tool-call count, and wall time. The dashboard charts token reduction from `website/public/benchmark/graph.json`; the visible axis runs from `-5%` to `100%`, baseline is `0%`, and hover details show the matching raw tokens, tool calls, and wall time.

## Structural Coverage

The structural benchmark loads a project's resident `Program`, builds the graph, and reports node and edge counts plus fair coverage: the share of symbol-bearing source files that have at least one resolved cross-file edge. Counts and coverage are deterministic. Timings are indicative and only meaningful on a quiet host.

Run it against any TypeScript project with a `tsconfig.json`:

```bash
node experimental/benchmark/graph/bench.mjs --project=/abs/path/to/project --runs=5
```

## Reproduce

Both harnesses start from a clone of this repository:

```bash
git clone https://github.com/samchon/ttsc.git
cd ttsc
corepack enable
pnpm install
```

### Structural harness

`experimental/benchmark/graph` loads the resident `Program` for a project, builds the graph, and reports the node and edge counts plus the fair coverage. The counts and coverage are deterministic; the timings are indicative and only meaningful on a quiet host.

```bash
node experimental/benchmark/graph/bench.mjs --project=/abs/path/to/project --runs=5
```

### Agent-cost harness

Baseline-only publication refresh:

```bash
node experimental/benchmark/graph.mjs --all --models=gpt-5.4-mini --arm=baseline --tools=baseline --prompt-family=all --runs=5 --reset
```

Codex trace audit:

```bash
node experimental/benchmark/graph/audit-codex-traces.mjs --dir=experimental/benchmark/.work/graph/<timestamp>
node experimental/benchmark/graph/audit-codex-traces.mjs --compare=<before>,<after>
node experimental/benchmark/graph/audit-codex-traces.mjs --self-test
```

`graph.mjs` writes `codex-trace-audit.json` automatically for Codex cells. The Codex audit records every exposed message, every shell/MCP call in timeline order, usage, reasoning token counts, strict exact avoidable output such as duplicate MCP calls and legacy inline evidence text, measured graph-replaceable shell-output surface, candidate MCP overfetch surfaces, zero-MCP and shell-fallback graph-arm traces, later-turn prompt replay exposure when multiple `turn.completed` events are visible, an input ledger comparing usage input tokens with visible trace material, and observed/replacement-lower-bound/candidate-ceiling savings against matching N=5 baseline medians from `website/public/benchmark/graph.json`. Hidden reasoning text is not emitted in the Codex stream, so the audit records the token ledger and explicitly marks the text unavailable. The ledger's unexplained input is an accounting gap, not proof of one hidden category. Use `--compare=<before>,<after>` with audit JSON files, suite reports, or suite directories to compare N=1 optimization smoke runs by the same message/tool/reasoning/savings fields without publishing them.

Comparator refresh after the baseline exists:

```bash
node experimental/benchmark/graph.mjs --all --models=gpt-5.4-mini --arm=graph --tools=ttsc-graph,codegraph,codebase-memory --prompt-family=all --runs=5
```

Use `--no-website` for local-only runs. The runner writes raw reports under `experimental/benchmark/.work/graph/<timestamp>/` and upserts website cells into `website/public/benchmark/graph.json`.
