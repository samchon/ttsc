# Regular expressions

Regular-expression rules from `eslint-plugin-regexp`. These rules check the structure of regex literals ‚Äî emptiness, uselessness, flag ordering, shorthand classes, Unicode support. Some rules duplicate (and supersede) the regex-related rules in [Core](./core); both ids exist so projects can keep the legacy ESLint names alongside the regexp-plugin variants. Dynamic `RegExp("...")` constructor strings are deferred until the lint engine has a shared literal-value helper.

Source: [`eslint-plugin-regexp`](https://github.com/ota-meshi/eslint-plugin-regexp) (MIT).

- `regexp/no-control-character` ‚Äî Reject ASCII control characters in regex literals.
- `regexp/no-dupe-characters-character-class` ‚Äî Reject duplicate literal characters inside simple regex character classes (`/[aa]/`).
- `regexp/no-empty-alternative` ‚Äî Reject empty alternatives in a disjunction (`/a||b/`), which silently match the empty string.
- `regexp/no-empty-capturing-group` ‚Äî Reject empty capturing groups such as `/()/`.
- `regexp/no-empty-character-class` ‚Äî Reject empty regex character classes (`[]`).
- `regexp/no-empty-group` ‚Äî Reject empty non-capturing groups such as `/(?:)/`.
- `regexp/no-empty-lookarounds-assertion` ‚Äî Reject empty lookaround assertions such as `/(?=)/` or `/(?!)/`.
- `regexp/no-misleading-unicode-character` ‚Äî Reject misleading Unicode characters in regex classes.
- `regexp/no-useless-character-class` ‚Äî Reject single-character character classes such as `/[x]/` ‚Äî `/x/` is equivalent.
- `regexp/no-useless-escape` ‚Äî Reject unnecessary escapes inside regex literals.
- `regexp/no-useless-flag` ‚Äî Reject regex flags that the literal does not exercise ‚Äî `i` on a pattern without case-variable characters, `m` without `^`/`$`, `s` without `.`, and similar dead flags on `g`/`y`.
- `regexp/no-useless-quantifier` ‚Äî Reject quantifiers that do not change the match ‚Äî constant-one counts (`/a{1}/`), `?` on patterns already matching the empty string, and quantifiers on non-consuming atoms.
- `regexp/no-useless-two-nums-quantifier` ‚Äî Reject equal min/max quantifiers (`/a{2,2}/`) in favor of `/a{2}/`.
- `regexp/no-zero-quantifier` ‚Äî Reject zero-repeat quantifiers (`/a{0}/`, `/a{0,0}/`) ‚Äî the atom never matches, so the quantifier is either dead code or a typo for `{1,‚Ä¶}`.
- `regexp/prefer-d` ‚Äî Prefer `\d` over `[0-9]` in regex literals.
- `regexp/prefer-plus-quantifier` ‚Äî Prefer `+` over `{1,}` in regex literals.
- `regexp/prefer-question-quantifier` ‚Äî Prefer `?` over `{0,1}` in regex literals.
- `regexp/prefer-star-quantifier` ‚Äî Prefer `*` over `{0,}` in regex literals.
- `regexp/prefer-w` ‚Äî Prefer `\w` over `[A-Za-z0-9_]` in regex literals.
- `regexp/require-unicode-regexp` ‚Äî Require regex literals to use the `u` or `v` flag, so Unicode-property escapes and surrogate-pair handling stay predictable.
- `regexp/require-unicode-sets-regexp` ‚Äî Require regex literals to use the `v` flag specifically ‚Äî the stricter Unicode-sets mode that enables set notation, string properties, and stricter escape rules on top of `u`.
- `regexp/sort-flags` ‚Äî Require regex flags to appear in canonical alphabetical order (`dgimsuvy`).
