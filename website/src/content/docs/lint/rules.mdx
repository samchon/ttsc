# Rules

A practical rule catalog by category. Each rule maps to an ESLint name where one exists. Severity in `lint.config.ts` is one of `"error"` (fails the build), `"warning"` (prints, exit code 0), `"off"` (skipped).

If you're skimming: the `error`-severity defaults at the bottom of this page are what most projects end up with.

> **Authoritative source:** the canonical rule-name union lives at [`packages/lint/src/structures/TtscLintRule.ts`](https://github.com/samchon/ttsc/tree/master/packages/lint/src/structures/TtscLintRule.ts). This page documents what each rule does and when to keep it on.

## Variable declarations

| Rule | Effect |
| ---- | ------ |
| `no-var` | Reject `var`. Use `let` or `const`. Autofixable. |
| `prefer-const` | When a `let` is never reassigned, demand `const`. Autofixable for single declarations. |

## Cypress

These rules inspect high-confidence Cypress and Mocha test shapes in TS/TSX specs, including `cy.*` command chains and `it` / `before` callbacks.

| Rule | Effect |
| ---- | ------ |
| `cypress/assertion-before-screenshot` | Require a Cypress assertion before `cy.screenshot()`. |
| `cypress/no-and` | Prefer `.should()` over `.and()` when starting Cypress assertion chains. |
| `cypress/no-assigning-return-values` | Reject assigning the return value of Cypress commands. |
| `cypress/no-async-before` | Reject async `before` and `beforeEach` callbacks. |
| `cypress/no-async-tests` | Reject async Cypress test callbacks. |
| `cypress/no-chained-get` | Reject chained `.get()` calls; use `.find()` from the previous subject. |
| `cypress/no-debug` | Reject `cy.debug()` and chained `.debug()` commands. |
| `cypress/no-force` | Reject `{ force: true }` on Cypress action commands. |
| `cypress/no-pause` | Reject `cy.pause()` and chained `.pause()` commands. |
| `cypress/no-unnecessary-waiting` | Reject numeric `cy.wait(...)` sleeps. |
| `cypress/no-xpath` | Reject deprecated `cy.xpath()` selectors. |
| `cypress/require-data-selectors` | Require statically known `cy.get()` selectors to target `data-*` attributes. |
| `cypress/unsafe-to-chain-command` | Reject chaining more commands after Cypress action commands. Option: `{ methods?: string[] }`. |

## Directive comments

These rules inspect `eslint-disable` / `eslint-enable` comments and the matching `lint-disable` aliases supported by `@ttsc/lint`.

| Rule | Effect |
| ---- | ------ |
| `eslint-comments/disable-enable-pair` | Require range `eslint-disable` directives to have a later matching `eslint-enable`. Option: `{ allowWholeFile?: boolean }`. |
| `eslint-comments/no-aggregating-enable` | Reject bare `eslint-enable` comments that re-enable named disables at once. |
| `eslint-comments/no-duplicate-disable` | Reject repeated disables for a rule that is already disabled. |
| `eslint-comments/no-restricted-disable` | Reject disables for configured protected rules. Option: `{ rules: string[] }`. |
| `eslint-comments/no-unlimited-disable` | Reject disable comments with no explicit rule list. |
| `eslint-comments/no-unused-disable` | Reject disable comments that suppress no diagnostic. |
| `eslint-comments/no-unused-enable` | Reject enable comments that do not re-enable anything. |
| `eslint-comments/no-use` | Reject lint directive comments entirely. Option: `{ allow?: string[] }`. |
| `eslint-comments/require-description` | Require directive comments to include a `--` description. |

## Regular expressions

The first `regexp/*` wave mirrors high-confidence `eslint-plugin-regexp`
checks for regex literals. Dynamic `RegExp("...")` constructor strings are
deferred until the lint engine has a shared literal-value helper.

| Rule | Effect |
| ---- | ------ |
| `regexp/no-control-character` | Reject control characters in regex literals. Alias of the older bare regex check. |
| `regexp/no-dupe-characters-character-class` | Reject duplicate literal characters in simple character classes. |
| `regexp/no-empty-alternative` | Reject empty alternatives such as `/a\|\|b/`. |
| `regexp/no-empty-capturing-group` | Reject empty capturing groups such as `/()/`. |
| `regexp/no-empty-character-class` | Reject empty character classes. Alias of `no-empty-character-class`. |
| `regexp/no-empty-group` | Reject empty non-capturing groups such as `/(?:)/`. |
| `regexp/no-empty-lookarounds-assertion` | Reject empty lookaround assertions such as `/(?=)/`. |
| `regexp/no-misleading-unicode-character` | Reject misleading Unicode characters in character classes. Alias of the older bare misleading-character check. |
| `regexp/no-useless-character-class` | Reject single literal character classes such as `/[x]/`. |
| `regexp/no-useless-escape` | Reject unnecessary regex escapes. Alias of `no-useless-escape` for regex literals. |
| `regexp/no-useless-flag` | Reject flags that do not affect the literal, such as `/\d+/i`. |
| `regexp/no-useless-quantifier` | Reject exact-one quantifiers such as `/a{1}/`. |
| `regexp/no-useless-two-nums-quantifier` | Reject equal min/max quantifiers such as `/a{2,2}/`. |
| `regexp/no-zero-quantifier` | Reject zero-repeat quantifiers such as `/a{0}/`. |
| `regexp/prefer-d` | Prefer `\d` over `[0-9]`. |
| `regexp/prefer-plus-quantifier` | Prefer `+` over `{1,}`. |
| `regexp/prefer-question-quantifier` | Prefer `?` over `{0,1}`. |
| `regexp/prefer-star-quantifier` | Prefer `*` over `{0,}`. |
| `regexp/prefer-w` | Prefer `\w` over `[A-Za-z0-9_]`. |
| `regexp/require-unicode-regexp` | Require the `u` or `v` flag. |
| `regexp/require-unicode-sets-regexp` | Require the `v` flag. |
| `regexp/sort-flags` | Require canonical flag order. |

## Architecture boundaries

These rules are inspired by `eslint-plugin-boundaries`, but `@ttsc/lint` implements a conservative TypeScript source-path subset. They inspect static `import` and `export ... from` module specifiers, resolve relative imports to real source files, and classify files with the rule option's `elements` array.

| Rule | Effect |
| ---- | ------ |
| `boundaries/element-types` | Enforce allowed dependency directions between configured source-path element types. |
| `boundaries/external` | Restrict external package imports by package/specifier pattern. |
| `boundaries/entry-point` | Require imports into an element to target that element's configured public entry files. |
| `boundaries/no-private` | Reject imports of configured private files from outside their element. |
| `boundaries/no-unknown` | Reject relative imports whose resolved source file matches no configured element. |

```ts
// lint.config.ts
export default {
  rules: {
    "boundaries/element-types": ["error", {
      elements: [
        { type: "app", pattern: "src/app/**" },
        { type: "domain", pattern: "src/domain/**", entry: "index.ts", private: "internal/**" },
      ],
      rules: [{ from: "app", disallow: "domain" }],
    }],
    "boundaries/entry-point": ["error", {
      elements: [
        { type: "domain", pattern: "src/domain/**", entry: "index.ts" },
      ],
    }],
    "boundaries/no-private": ["error", {
      elements: [
        { type: "domain", pattern: "src/domain/**", private: "internal/**" },
      ],
    }],
    "boundaries/no-unknown": ["error", {
      elements: [
        { type: "app", pattern: "src/app/**" },
        { type: "domain", pattern: "src/domain/**" },
      ],
    }],
    "boundaries/external": ["error", { disallow: ["@legacy/sdk"] }],
  },
};
```

These diagnostics are not autofixable. A boundary violation usually needs a public API, a dependency inversion, or a config decision; rewriting the import path automatically would be unsafe.

## JSX accessibility

The `jsx-a11y/*` rules mirror TSX-applicable checks from `eslint-plugin-jsx-a11y`
for intrinsic JSX elements. They are diagnostic-only in this release. Component
alias settings and router-specific anchor settings are deferred, so these rules
focus on static tag, attribute, role, and child-content checks that the TypeScript
AST can prove locally.

| Rule | Effect |
| ---- | ------ |
| `jsx-a11y/alt-text` | Require text alternatives on `img`, `area`, image inputs, and `object`. |
| `jsx-a11y/anchor-has-content` | Reject empty anchors without text or an accessible label. |
| `jsx-a11y/anchor-is-valid` | Reject missing, empty, hash-only, and `javascript:` anchor href values. |
| `jsx-a11y/aria-activedescendant-has-tabindex` | Require `tabIndex` on elements with `aria-activedescendant`. |
| `jsx-a11y/aria-props` | Reject unknown `aria-*` attributes. |
| `jsx-a11y/aria-proptypes` | Validate statically known ARIA boolean, tristate, and numeric values. |
| `jsx-a11y/aria-role` | Reject unknown ARIA role tokens. |
| `jsx-a11y/aria-unsupported-elements` | Reject ARIA attributes and roles on unsupported elements such as `meta`. |
| `jsx-a11y/autocomplete-valid` | Reject unknown literal `autocomplete` / `autoComplete` tokens. |
| `jsx-a11y/click-events-have-key-events` | Require keyboard handlers alongside `onClick` on non-interactive elements. |
| `jsx-a11y/control-has-associated-label` | Require interactive controls to have an accessible label. |
| `jsx-a11y/heading-has-content` | Reject empty headings. |
| `jsx-a11y/html-has-lang` | Require `lang` on `html`. |
| `jsx-a11y/iframe-has-title` | Require non-empty iframe titles. |
| `jsx-a11y/img-redundant-alt` | Reject redundant words such as image, photo, and picture in image alt text. |
| `jsx-a11y/interactive-supports-focus` | Require focusability for elements with interactive roles. |
| `jsx-a11y/label-has-associated-control` | Require labels to reference or wrap a control. |
| `jsx-a11y/label-has-for` | Compatibility alias for label association checks. |
| `jsx-a11y/lang` | Validate statically known `lang` values. |
| `jsx-a11y/media-has-caption` | Require caption or subtitle tracks under `audio` and `video`. |
| `jsx-a11y/mouse-events-have-key-events` | Pair mouseover/mouseout handlers with focus/blur handlers. |
| `jsx-a11y/no-access-key` | Reject `accessKey`. |
| `jsx-a11y/no-aria-hidden-on-focusable` | Reject `aria-hidden` on focusable elements. |
| `jsx-a11y/no-autofocus` | Reject `autoFocus` / `autofocus`. |
| `jsx-a11y/no-distracting-elements` | Reject `blink` and `marquee`. |
| `jsx-a11y/no-interactive-element-to-noninteractive-role` | Reject non-interactive roles on native interactive elements. |
| `jsx-a11y/no-noninteractive-element-interactions` | Reject interaction handlers on known non-interactive elements. |
| `jsx-a11y/no-noninteractive-element-to-interactive-role` | Reject interactive roles on known non-interactive elements. |
| `jsx-a11y/no-noninteractive-tabindex` | Reject `tabIndex` on non-interactive elements. |
| `jsx-a11y/no-redundant-roles` | Reject explicit roles that duplicate native semantics. |
| `jsx-a11y/no-static-element-interactions` | Require a role on static elements with interaction handlers. |
| `jsx-a11y/prefer-tag-over-role` | Prefer native tags over `div` / `span` plus equivalent roles. |
| `jsx-a11y/role-has-required-aria-props` | Require ARIA properties mandated by a role. |
| `jsx-a11y/role-supports-aria-props` | Reject ARIA properties unsupported by a role. |
| `jsx-a11y/scope` | Restrict `scope` to `th`. |
| `jsx-a11y/tabindex-no-positive` | Reject positive `tabIndex` values. |

## React Hooks

These rules mirror `eslint-plugin-react-hooks` names. The first native slice implements the core Hook
rules and an AST-local subset of the React Compiler-era checks; compiler rules that need whole-program
React Compiler IR remain deferred.

| Rule | Effect |
| ---- | ------ |
| `react-hooks/rules-of-hooks` | Reject Hooks outside components/custom Hooks, inside nested callbacks, or in conditional/loop control flow. |
| `react-hooks/exhaustive-deps` | Reject high-confidence missing identifier dependencies in `useEffect`, `useLayoutEffect`, `useInsertionEffect`, `useMemo`, and `useCallback`. |
| `react-hooks/component-hook-factories` | Reject nested component/Hook factories that call Hooks. |
| `react-hooks/set-state-in-render` | Reject `useState`/`useReducer` setters called during render. |
| `react-hooks/set-state-in-effect` | Reject synchronous setter calls inside `useEffect`. |
| `react-hooks/immutability` | Reject local prop mutation in components and Hooks. |
| `react-hooks/refs` | Reject reading or writing `ref.current` during render. |
| `react-hooks/use-memo` | Reject block-bodied `useMemo` callbacks that do not return a value. |

Deferred `eslint-plugin-react-hooks@7.1.1` targets include `config`, `error-boundaries`, `gating`,
`globals`, `incompatible-library`, `preserve-manual-memoization`, `purity`, `static-components`,
`unsupported-syntax`, `void-use-memo`, and other off-by-default compiler diagnostics such as
`memo-dependencies`, `memoized-effect-dependencies`, and `exhaustive-effect-dependencies`.

## Throw and control flow

## Functional Policy

The `functional/*` family is an opt-in strictness pack for projects that want
immutable, expression-oriented TypeScript. These rules are intentionally not
core defaults because they encode a programming style, not just bug prevention.
They are diagnostic-only; `ttsc fix` does not rewrite architecture or control
flow choices automatically.

| Rule | Effect |
| ---- | ------ |
| `functional/functional-parameters` | Reject rest parameters, `arguments`, and optionally zero-parameter functions. |
| `functional/immutable-data` | Reject writes through object/array members and mutable collection methods. |
| `functional/no-class-inheritance` | Reject class inheritance and abstract classes. |
| `functional/no-classes` | Reject class declarations and expressions. |
| `functional/no-conditional-statements` | Reject `if` and `switch` statements. |
| `functional/no-expression-statements` | Reject expression statements used for side effects. |
| `functional/no-let` | Reject `let` declarations. |
| `functional/no-loop-statements` | Reject imperative loop statements. |
| `functional/no-mixed-types` | Reject type/interface declarations that mix member shapes. |
| `functional/no-promise-reject` | Reject `Promise.reject(...)`. |
| `functional/no-return-void` | Reject void returns and void-returning declarations. |
| `functional/no-this-expressions` | Reject `this` expressions. |
| `functional/no-throw-statements` | Reject `throw` statements. |
| `functional/no-try-statements` | Reject `try`/`catch`/`finally` statements. |
| `functional/prefer-immutable-types` | Prefer readonly/immutable type annotations. |
| `functional/prefer-property-signatures` | Prefer function-property signatures over method signatures. |
| `functional/prefer-readonly-type` | Require readonly array, tuple, and property type syntax. |
| `functional/prefer-tacit` | Report simple one-argument forwarding wrappers. |

## A common starter set

Most projects start here. Add and remove as you go.

```ts
// lint.config.ts
import type { ITtscLintConfig } from "@ttsc/lint";

export default {
  rules: {
    // Variables
    "no-var": "error",
    "prefer-const": "error",

    // Comparison
    "eqeqeq": "error",
    "no-self-compare": "error",

    // Suggestions
    "no-explicit-any": "warning",
    "prefer-as-const": "error",
    "object-shorthand": "error",
    "no-useless-rename": "error",

    // Imports
    "no-import-type-side-effects": "error",
    "consistent-type-imports": "error",

    // Misc safety
    "no-debugger": "error",
    "no-throw-literal": "error",
    "await-thenable": "error",

    // Format (see /docs/lint/format)
    "format/semi": "error",
    "format/quotes": ["error", { prefer: "double" }],
    "format/trailing-comma": "error",
    "format/print-width": ["error", { printWidth: 80 }],
  },
} satisfies ITtscLintConfig;
```

## Autofix coverage at a glance

`ttsc fix` autofixes a subset. Native fixers cover:

- `no-var`, single-declaration `prefer-const`, ESLint-safe `eqeqeq`
- `no-wrapper-object-types` (primitives only; `Object` is detection-only)
- `prefer-as-const`, `no-useless-rename`, `object-shorthand`, `no-extra-non-null-assertion`
- `no-unnecessary-type-constraint`, `prefer-namespace-keyword`, `no-useless-escape`
- `no-import-type-side-effects` (hoists inline `type` modifiers)
- `await-thenable` (type-aware — drops `await` on non-thenable operands)
- Every format rule
