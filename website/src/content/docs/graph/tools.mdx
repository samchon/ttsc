# Tools

`@ttsc/graph` exposes one MCP tool: `inspect_typescript_project_graph_before_answering`. It is a narrowing engine over the checker-resolved graph. The model first states why graph evidence is needed, writes `draft.reason` before `draft.type`, reviews that draft for overfetch or wrong-tool use, then submits one request branch.

Every node and edge is resolved by the TypeScript checker, so the graph is a checker-resolved fact map end to end. The MCP return value is always one object: `{ result: ... }`. The `result` property is the union branch, and `result.type` matches the selected request branch.

Use `escape` when the review shows the graph is the wrong evidence source, or the previous result is enough. That branch returns without running a graph lookup or traversal.

## inspect_typescript_project_graph_before_answering

```ts
declare function inspect_typescript_project_graph_before_answering(
  props: ITtscGraphApplication.IProps,
): ITtscGraphApplication.IResult;

namespace ITtscGraphApplication {
  interface IProps {
    /** User's TypeScript code question, restated for this graph call. */
    question: string;
    /** Why the resident graph is or is not the next evidence source. */
    graphNeed: string;
    /** First-pass operation choice before final arguments are filled. */
    draft: {
      reason: string;
      type:
        | ITtscGraphEntrypoints.IRequest["type"]
        | ITtscGraphLookup.IRequest["type"]
        | ITtscGraphTrace.IRequest["type"]
        | ITtscGraphDetails.IRequest["type"]
        | ITtscGraphOverview.IRequest["type"]
        | ITtscGraphEscape.IRequest["type"];
    };
    /** Review the draft for overfetch, shell fallback, web lookup, or wrong scope. */
    review: string;
    /** Exactly one final request branch. */
    request:
      | ITtscGraphEntrypoints.IRequest
      | ITtscGraphLookup.IRequest
      | ITtscGraphTrace.IRequest
      | ITtscGraphDetails.IRequest
      | ITtscGraphOverview.IRequest
      | ITtscGraphEscape.IRequest;
  }

  interface IResult {
    result:
      | ITtscGraphEntrypoints
      | ITtscGraphLookup
      | ITtscGraphTrace
      | ITtscGraphDetails
      | ITtscGraphOverview
      | ITtscGraphEscape;
  }
}
```

## Request Branches

`ITtscGraphEntrypoints.IRequest` returns a compact shortlist for behavior-specific code questions: ranked symbols, direct mentions, small dependency orientation, and follow-up handles. It is the source-free replacement for broad search.

`ITtscGraphLookup.IRequest` finds a class, method, function, property, type, or file concept when you do not already have its handle. Keep the default limit unless a previous result was ambiguous.

`ITtscGraphTrace.IRequest` follows call, type, or dependency flow. Use `to` for "how A reaches B" questions instead of widening `maxNodes`; use `reverse` for callers and impact checks.

`ITtscGraphDetails.IRequest` inspects selected handles. By default it returns signatures, members, direct calls, and direct types. Set `source: true` only for one or two decisive leaf bodies; split source reads from neighbor maps.

`ITtscGraphOverview.IRequest` returns a source-free architecture map: counts, layers, hotspots, and public API symbols.

`ITtscGraphEscape.IRequest` returns a small no-op result when the review finds that no graph operation should run.

```ts
namespace ITtscGraphEntrypoints {
  interface IRequest {
    type: "entrypoints";
    query: string;
    limit?: number; // default 5
    neighbors?: number; // default 1
  }
}

namespace ITtscGraphLookup {
  interface IRequest {
    type: "lookup";
    query: string;
    limit?: number; // default 12
  }
}

namespace ITtscGraphTrace {
  interface IRequest {
    type: "trace";
    from: string;
    to?: string;
    direction?: "forward" | "reverse" | "impact";
    focus?: "all" | "execution" | "types";
    maxDepth?: number;
    maxNodes?: number;
  }
}

namespace ITtscGraphDetails {
  interface IRequest {
    type: "details";
    handles: string[];
    neighbors?: boolean;
    neighborLimit?: number;
    source?: boolean;
    lineNumbers?: boolean;
  }
}

namespace ITtscGraphOverview {
  interface IRequest {
    type: "overview";
    aspect?: "all" | "layers" | "hotspots" | "publicApi";
  }
}

namespace ITtscGraphEscape {
  interface IRequest {
    type: "escape";
    reason: string;
    nextStep?: string;
  }
}
```

## Result Branches

The MCP result object itself is not a union. Its `result` property is.

```ts
interface ITtscGraphApplicationResult {
  result:
    | ITtscGraphEntrypoints
    | ITtscGraphLookup
    | ITtscGraphTrace
    | ITtscGraphDetails
    | ITtscGraphOverview
    | ITtscGraphEscape;
}

interface ITtscGraphEntrypoints {
  type: "entrypoints";
  query: string;
  hits: EntrypointsHit[];
  mentions: EntrypointsMention[];
  neighborhood: EntrypointsNeighborhood[];
  next: { details: string[]; traceFrom: string[] };
  truncated?: boolean;
}

interface ITtscGraphLookup {
  type: "lookup";
  hits: LookupHit[];
  next: { details: string[]; traceFrom: string[] };
}

interface ITtscGraphTrace {
  type: "trace";
  start?: TraceNode;
  direction: string;
  hops: TraceHop[];
  reached: TraceNode[];
  truncated: boolean;
}

interface ITtscGraphDetails {
  type: "details";
  nodes: DetailsNode[];
  unknown: string[];
}

interface ITtscGraphOverview {
  type: "overview";
  project: string;
  counts: OverviewCounts;
}

interface ITtscGraphEscape {
  type: "escape";
  skipped: true;
  reason: string;
  nextStep?: string;
}
```

The concrete payloads carry symbol coordinates, edge evidence, sourceSpan anchors, and stable handles for follow-up graph calls.

## Node Kinds

Symbol kinds are declarations the checker owns: `file`, `package`, `namespace`, `module`, `function`, `class`, `interface`, `type`, `enum`, `variable`, `method`, `property`, `parameter`. `external_symbol` is a dependency-boundary leaf. The graph does not interpret framework conventions; the checker-resolved decorator facts on a node are left for an agent to read.

## Edge Kinds

| Kind           | Meaning                                     |
| -------------- | ------------------------------------------- |
| `contains`     | File/class/namespace ownership of a member. |
| `exports`      | A file exports a symbol.                    |
| `imports`      | A module imports another.                   |
| `calls`        | A runtime call.                             |
| `instantiates` | A `new T()` construction.                   |
| `accesses`     | A property/accessor read or write.          |
| `type_ref`     | A type-position reference.                  |
| `extends`      | Class/interface `extends`.                  |
| `implements`   | Class `implements`.                         |
| `overrides`    | A member overrides a base member.           |
| `decorates`    | A decorator applied to a declaration.       |
| `renders`      | A JSX component use.                        |
| `tests`        | A test exercises a symbol.                  |
