# Tools

`@ttsc/graph` exposes one MCP tool: `query`. It is a narrowing engine over the checker-resolved graph. The model first states why graph evidence is needed, writes `draft.reason` before `draft.type`, reviews that draft for overfetch or wrong-tool use, then submits one request branch.

Every node and edge is resolved by the TypeScript checker, so the graph is a checker-resolved fact map end to end. The MCP return value is always one object: `{ result: ... }`. The `result` property is the union branch, and `result.type` matches the selected request branch.

Use `escape` when the review shows the graph is the wrong evidence source, or the previous result is enough. That branch returns without running a graph lookup or traversal.

## query

The tool contract is `ITtscGraphApplication.ts`. Its request branches are the files listed below.

## Request Branches

`ITtscGraphEntrypoints.IRequest` returns a compact shortlist for behavior-specific code questions: ranked symbols, direct mentions, small dependency orientation, and follow-up handles. It is the source-free replacement for broad search.

`ITtscGraphLookup.IRequest` finds a class, method, function, property, type, or file concept when you do not already have its handle. Keep the default limit unless a previous result was ambiguous.

`ITtscGraphTrace.IRequest` follows call, type, or dependency flow. Use `to` for "how A reaches B" questions instead of widening `maxNodes`; use `reverse` for callers and impact checks.

`ITtscGraphDetails.IRequest` inspects selected handles. It returns signatures, members, direct calls, direct types, dependency neighbors, and sourceSpan anchors without implementation text.

`ITtscGraphOverview.IRequest` returns a source-free architecture map: counts, layers, hotspots, and public API symbols.

`ITtscGraphEscape.IRequest` returns a small no-op result when the review finds that no graph operation should run.

## Result Branches

The MCP result object itself is not a union. Its `result` property is.

The concrete payloads carry symbol coordinates, edge evidence ranges, sourceSpan anchors, and stable handles for follow-up graph calls.

## Node Kinds

Symbol kinds are declarations the checker owns: `file`, `package`, `namespace`, `module`, `function`, `class`, `interface`, `type`, `enum`, `variable`, `method`, `property`, `parameter`. `external_symbol` is a dependency-boundary leaf. The graph does not interpret framework conventions; the checker-resolved decorator facts on a node are left for an agent to read.

## Edge Kinds

| Kind           | Meaning                                     |
| -------------- | ------------------------------------------- |
| `contains`     | File/class/namespace ownership of a member. |
| `exports`      | A file exports a symbol.                    |
| `imports`      | A module imports another.                   |
| `calls`        | A runtime call.                             |
| `instantiates` | A `new T()` construction.                   |
| `accesses`     | A property/accessor read or write.          |
| `type_ref`     | A type-position reference.                  |
| `extends`      | Class/interface `extends`.                  |
| `implements`   | Class `implements`.                         |
| `overrides`    | A member overrides a base member.           |
| `decorates`    | A decorator applied to a declaration.       |
| `renders`      | A JSX component use.                        |
| `tests`        | A test exercises a symbol.                  |
