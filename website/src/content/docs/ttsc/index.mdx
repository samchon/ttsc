# TTSC

The compiler, the runtime, the language server, and the bundler adapter — one toolchain, one plugin contract, one source of truth (`tsconfig.json`).

```bash
npx ttsc           # build · check · watch · fix · format
npx ttsx file.ts   # run TypeScript with a real type-check
```

## In this chapter

- [Compile](/docs/ttsc/compile) — `ttsc` build / check / watch / fix / format. Day-to-day commands and the full flag reference.
- [Execute](/docs/ttsc/execute) — `ttsx`. The typed replacement for `tsx` and `ts-node`.
- [Bundler integration](/docs/ttsc/bundler) — `@ttsc/unplugin`. Same plugin pass inside Vite, Webpack, Rollup, esbuild, Rspack, Rolldown, Next.js, Farm, Bun.

For the small first-party plugins shipped in this repo — [@ttsc/banner](/docs/plugins/banner), [@ttsc/paths](/docs/plugins/paths), [@ttsc/strip](/docs/plugins/strip) — see their own pages under Other Plugins. For `@ttsc/lint`, see [Lint](/docs/lint). Third-party plugins like [typia](https://typia.io) install through the same `compilerOptions.plugins` entry; check their own docs.

## How they fit together

Every `ttsc` build is one pass over your project:

1. Read `tsconfig.json`.
2. Type-check the project.
3. Run every plugin in `compilerOptions.plugins`, in order.
4. Emit JS + `.d.ts` (or skip, with `--noEmit`).

`ttsx` does steps 1-3, then runs the result instead of emitting. `@ttsc/unplugin` runs the same plugin pass inside a bundler. The VSCode extension does steps 1-3 continuously as you type and surfaces the diagnostics in the editor.

You configure plugins **once**, in `tsconfig.json`. Every entry point sees them.
