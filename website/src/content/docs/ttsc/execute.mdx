# Execute (ttsx)

`ttsx` is the runner form of `ttsc`: it checks a TypeScript entry file, applies your configured plugins, then runs the file.

## Run a script

```bash
npx ttsx src/index.ts
```

The script starts only when the check passes. Type errors stop execution, and so do plugin errors such as `@ttsc/lint` errors. Warnings do not stop execution.

Put script arguments after `--`:

```bash
npx ttsx src/server.ts -- --port 3000
```

Pick a tsconfig with `-P`:

```bash
npx ttsx -P tsconfig.scripts.json src/seed.ts
```

Preload a module with `-r`, the same as Node's `--require`:

```bash
npx ttsx -r dotenv/config src/index.ts
```

## Options

| Flag                     | Meaning                              |
| ------------------------ | ------------------------------------ |
| `-P, --project <file>`   | Use a specific `tsconfig.json`.      |
| `--cwd <dir>`            | Run from another working directory.  |
| `--cache-dir <dir>`      | Use a specific cache directory.      |
| `--binary <path>`        | Use a specific TypeScript-Go binary. |
| `--no-plugins`           | Run without ttsc plugins.            |
| `-r, --require <module>` | Preload a module. Repeatable.        |
| `-h, --help`             | Show help.                           |
| `-v, --version`          | Print the version.                   |

Run `npx ttsx --help` for every option.
