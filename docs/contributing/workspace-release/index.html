Workspace Release

This page is for ttsc maintainers publishing the @ttsc/* npm packages. If you are writing your own plugin and looking for the publish flow for a plugin you own, go to Authoring → Publishing instead.

Local Loop

pnpm install
pnpm format
pnpm build
pnpm test

Use the focused current-platform build while iterating on native or launcher code:

pnpm run build:current
pnpm run test:features -- --include=ttsc
pnpm --dir packages/ttsc go:vet

The root pnpm test script builds the current platform package, type-checks the test workspace, then runs the tests/test-* feature packages. Production package manifests do not own test scripts; test execution is centralized under tests.

Go-specific maintenance uses separate commands:

pnpm --dir packages/ttsc go:vet
pnpm run coverage:go
node scripts/test-go-transformer.cjs
node scripts/test-go-lint.cjs
node scripts/test-go-utility-plugins.cjs

Tarball Smoke

Build local tarballs:

pnpm run package:tgz

Run packed-package install checks:

pnpm run experimental:install
pnpm run experimental:unplugin
pnpm run experimental

experimental:install validates the packed ttsc package, the current platform package, bundled Go, first-party utility plugins, ttsc --version, ttsc --emit, and ttsx. experimental:unplugin validates published-package imports and bundler adapters from a clean consumer project.

Platform Packages

The ttsc package publishes platform helpers as optional dependencies:

  • @ttsc/linux-x64
  • @ttsc/linux-arm
  • @ttsc/linux-arm64
  • @ttsc/darwin-x64
  • @ttsc/darwin-arm64
  • @ttsc/win32-x64
  • @ttsc/win32-arm64

Each platform package contains the native helper and a bundled Go SDK for source-plugin builds. scripts/build-platform-package.cjs selects the bundled Go binary in this order: TTSC_GO_BINARY first (absolute path to a Go binary), then TTSC_GO_ROOT_{OS}_{ARCH}, then TTSC_GO_ROOT, then a download of the configured Go version when network access is available. TTSC_GO_VERSION overrides the bundled Go version.

Publish

The release workflow runs on pushed tags and executes:

pnpm run package:latest --no-git-checks --provenance

The root package:latest script builds platform packages first, publishes all non-ttsc packages, then publishes ttsc last so its optional dependencies are already available.

Adjacent root scripts:

  • pnpm run package:next (bash next.bash) — pre-release pipeline for next dist-tag publishes.
  • pnpm run release (bumpp -r) — recursive version bump across the workspace before tagging.

Release-Time Blockers

Three checks in .github/workflows/release.yml will block publish:

  1. WASM artifact size minimum (release.yml:28-35). Publish aborts if packages/wasm/dist/ttsc.wasm is under 10 MB — a smaller file means the WASM build produced a stub. Re-run pnpm --filter @ttsc/wasm build locally and confirm the artifact is intact before re-tagging.
  2. @ttsc/lint linthost tarball assertion (release.yml:36-39). The workflow pnpm packs @ttsc/lint and greps for linthost/ in the tarball; this check exists because of commit 2b3d37b (ci(release): pack @ttsc/lint from package dir), which fixed a regression where the lint engine source was missing from the published tarball. If the grep fails, check packages/lint/package.json#files for linthost/.
  3. Post-publish WASM smoke (release.yml:42-58). After publish, a fresh job runs npm install @ttsc/wasm@<just-published> from the public registry and verifies the binary is reachable. A failure here means the publish landed but the package is not consumable; investigate registry propagation before declaring the release good.
